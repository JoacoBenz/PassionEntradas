1A.Sincronización con un proveedor — obtención de datos

Cómo entra el sistema, se asegura la sesión y clasifica cada evento (Inmediato, On Request, u omitir).

flowchart TD
  s(["Inicio del ciclo"]) --> q1{"¿Sesión válida?"}
  q1 -->|"Sí"| L["Traer lista de eventos"]
  q1 -->|"No"| lg["Iniciar login"]
  lg --> q3{"¿Login correcto?"}
  q3 -->|"No"| bk["Backoff y reintento"]
  q3 -->|"Sí"| L
  L --> D["Por cada evento: traer detalle"]
  D --> q4{"¿Trae sectores con precio?"}
  q4 -->|"Sí"| q5{"¿Con stock disponible?"}
  q5 -->|"Sí"| inm["Marcar como Inmediato"]
  q5 -->|"No"| onr["Marcar como On Request"]
  q4 -->|"No"| q6{"¿Qué es?"}
  q6 -->|"Es On Request"| req["Fila a consultar"]
  q6 -->|"Servicio extra"| sk["Omitir"]
  q6 -->|"Sin datos"| inc["Marcar ciclo incompleto"]
  inm --> nx(["Continúa en la Parte B"])
  onr --> nx
  req --> nx
  classDef term fill:#0f172a,stroke:#0f172a,color:#ffffff;
  class s,nx term;

1B.Sincronización — publicación segura (anti-vaciado)

La regla crítica: nunca se vacía el catálogo por un error. Solo se marcan ausentes si el ciclo fue completo.

flowchart TD
  a(["Datos del ciclo"]) --> g{"Anti-vaciado: ¿0 ítems o caída mayor al umbral?"}
  g -->|"Sí, sospechoso"| ab(["Abortar: conservar catálogo anterior"])
  g -->|"No"| mk["Aplicar la comisión del agente: proveedor + categoría"]
  mk --> up["Guardar y actualizar en la base"]
  up --> q{"¿El ciclo fue completo?"}
  q -->|"Sí"| ma["Marcar ausentes como no disponibles"]
  q -->|"No"| nm["No marcar ausentes — evita falsos agotados"]
  ma --> done(["Fin del ciclo"])
  nm --> done
  classDef term fill:#0f172a,stroke:#0f172a,color:#ffffff;
  classDef bad fill:#fee2e2,stroke:#dc2626,color:#7f1d1d;
  class a,done term;
  class ab bad;

2.Cálculo de precio — comisión del agente y moneda

Cómo se arma el precio de venta: conversión a USD si hace falta, y la regla de comisión del agente más específica que aplique.

flowchart TD
  s(["Precio del proveedor"]) --> q1{"¿La moneda es USD?"}
  q1 -->|"No"| cv["Convertir a USD con la cotización del día"]
  q1 -->|"Sí"| q2{"¿Hay regla para proveedor + categoría?"}
  cv --> q2
  q2 -->|"Sí"| r1["Usar ese porcentaje"]
  q2 -->|"No"| q3{"¿Hay porcentaje del proveedor?"}
  q3 -->|"Sí"| r2["Usar el porcentaje del proveedor"]
  q3 -->|"No"| r3["Usar el porcentaje global por defecto"]
  r1 --> c["Precio de venta = precio + margen"]
  r2 --> c
  r3 --> c
  c --> o(["Precio de venta en USD"])
  classDef term fill:#0f172a,stroke:#0f172a,color:#ffffff;
  class s,o term;
Ejemplos de comisión del agente (a definir con el cliente): Passion Events · Fútbol → +20%  ·  Passion Events · F1 → +15%  ·  Passion Events · Conciertos → +25%  ·  Default del proveedor → +18%  ·  Default global → +20%.

El circuito de una entrada "a consultar": desde que la agencia la pide hasta el resultado.

4.Alta de un proveedor — módulo aparte

Cada proveedor nuevo entra por su propio módulo, que trae los datos y los conecta directo a la base —como Passion Events hoy.

flowchart TD
  s(["Nuevo proveedor"]) --> m["Módulo aparte (conector) — trae sus datos"]
  m --> n["Los traduce al formato común"]
  n --> rg["Aplica la comisión del agente del proveedor"]
  rg --> db["Se conecta directo a la base de datos"]
  db --> lv(["Proveedor activo en el catálogo"])
